MLX Voice Notes 需求规格说明书 (PRD)
1. 项目概述
1.1 产品定义
一款基于 Apple Silicon 加速的 macOS 原生音频创作工具，旨在为普通创作者（短视频博主、播客主）提供极简、高效的“文案转语音”工作流。支持多人对话识别、声音克隆，并采用“本地开源为主，云端 API 为辅”的混合架构。

1.2 核心价值
隐私与成本： 优先利用本地 MLX 框架进行推理，零成本且无需联网。

原生体验： 采用 SwiftUI 开发，符合 macOS 用户习惯的简洁笔记型交互。

生产力： 自动解析剧本文案，一键生成高质量多角色音频。

2. 功能详细说明
2.1 智能编辑器与文案解析
笔记型交互： 采用类似 Notion 的简洁界面，支持文本创作与编辑。

自动角色识别： * 支持解析 角色名：内容 或 [角色名]内容 格式。

软件自动将识别到的角色与音色库匹配，并在编辑器中高亮显示角色标签（Capsule 视觉）。

实时预览： 支持单句播放，点击段落旁按钮即时生成音频流。

局部调整： 允许选中特定文本重新生成，支持多版本结果对比。

2.2 推理引擎 (混合模式)
本地 MLX 引擎 (Primary)： * 集成 mlx-audio-swift 框架。

动态分段： 软件自动读取 Mac 硬件配置（RAM），动态调整长文本切片大小（8GB 内存压缩字数，32GB+ 增加字数），过程对用户透明。

云端 API 引擎 (Secondary)： * 预留第三方 API 接口（如 OpenAI, Azure 等）。

成本控制： 默认关闭，需用户手动开启；发送请求前显示预计字符消耗和费用；支持生成结果的本地缓存，避免重复扣费。

2.3 角色与音色管理 (Speaker Hub)
声音克隆： 支持导入 5-30s 参考音频或实时录音进行零样本（Zero-shot）克隆。

音色库： 管理系统预设音色与用户克隆音色，支持一键更换文案中对应的音色。

纯净人声： 初始版本仅支持人声输出，但架构需预留 BGM 背景音轨空间。

2.4 模型仓库 (Model Manager)
可视化管理： 提供模型下载列表，包含模型描述、体积及适用硬件建议。

完善的下载系统： 支持显示进度条、实时下载速度、断点续传、删除与更新管理。

2.5 导出与联动
格式支持： 导出 .mp3 格式，支持批量任务队列。

开放接口： * CLI 模式： 允许通过终端指令调用。

REST API： 本地启动 HTTP Server，供外部脚本调用。

持久化： 所有设置（包括 API 开关、模型选择）具备记忆性，软件重启后自动载入。

3. 用户界面 (UI) 设计规范
风格： 纯原生 SwiftUI 视觉，支持毛玻璃效果、深色模式、SF Symbols。

布局： * 左侧栏： 笔记列表、模型仓库、音色库管理。

主视窗： 笔记编辑器、生成进度显示。

底栏： 硬件资源占用情况、API 消耗预估、分段策略显示。

4. 技术架构建议
开发环境： Xcode 15+ / Swift 5.9+ / macOS 14+。

核心库： mlx-audio-swift (Blaizzy)。

网络： URLSession (Background) 处理模型下载。

存储： Core Data 或 Realm 存储笔记与纠音配置（预留）；UserDefaults 存储系统开关。

API 服务： 轻量级 Swift Web 服务器（如 Swifter）。

5. 开发路线图 (MVP)
Phase 1： 搭建 UI 框架，实现本地 mlx-audio-swift 桥接，支持单音色文本转语音。

Phase 2： 实现文案正则解析，支持多角色标签和声音克隆功能。

Phase 3： 完善模型下载管理系统，加入动态分段逻辑。

Phase 4： 接入第三方 API 模块及成本控制逻辑，开放 CLI/REST 接口。
