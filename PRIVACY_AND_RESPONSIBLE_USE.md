# Privacy and Responsible Use

MLX VibeVoice is designed as a local-first voice generation tool.

## Local Data

By default, the app stores user data on the user's Mac, including:

- scripts,
- generated audio,
- reference voice audio,
- reusable voice assets,
- model cache metadata,
- local app settings.

The app should not upload scripts, generated audio, or reference audio unless a
future cloud feature is explicitly added and clearly disclosed to the user.

## Voice Consent

Only create or use a reference voice when you own the audio or have permission
from the speaker. Do not use the app to impersonate another person, mislead an
audience, commit fraud, harass people, or violate platform rules or applicable
law.

## Publishing Generated Audio

Users are responsible for the content they generate and publish. Before sharing
or monetizing generated audio, verify that:

- the script content is lawful and allowed by the target platform,
- the reference voice is authorized,
- the model license allows the intended use,
- any required disclosure or attribution is included.

## Repository Hygiene

Do not commit user data, generated media, model weights, credentials, or local
SwiftData stores to the repository.
