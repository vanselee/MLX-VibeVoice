# Phase 0 Validation Plan

Phase 0 validates technical feasibility before low-fidelity UI prototyping or Phase 1 app development.

## Minimum Pass Criteria

- Swift-native path runs at least one Chinese TTS model.
- 1000 Chinese characters generate reliably and export to WAV.
- Multi-role script can be split, generated by role, and stitched.
- Final stitched WAV is 24kHz, 16-bit PCM, mono.

## Key Non-Blocking Validation

- Python/CLI `mlx-audio` local voice cloning feasibility.
- If local cloning fails in both Swift and Python/CLI paths, record the reason and mark Phase 3 cloning as delayed/experimental. Do not automatically switch to cloud cloning.

## Tasks

1. Verify local development environment.
   - Confirm full Xcode installation.
   - Confirm macOS 14+ and Apple Silicon.
   - Confirm Swift toolchain.

2. Run `mlx-audio-swift` sample.
   - Record setup steps.
   - Record model used.
   - Record errors and workarounds.

3. Build a minimal TTS spike.
   - Input: 200 Chinese characters.
   - Output: WAV.
   - Record generation time and memory use.

4. Validate long text.
   - Input: 1000 Chinese characters.
   - Test sentence-based segmentation.
   - Record failures, peak memory, and output quality.

5. Validate multi-role stitching.
   - Input: 3 roles, 10 dialogue turns.
   - Split by role.
   - Generate each segment.
   - Resample to 24kHz if needed.
   - Stitch into 16-bit PCM mono WAV.

6. Validate audio post-processing.
   - Decide LUFS measurement implementation.
   - Target +/- 1 LUFS against ITU-R BS.1770-4 reference behavior.
   - Validate 20ms segment fades and 80ms head/tail fades.

7. Validate Python/CLI local cloning path.
   - Test only after Swift-native path is understood.
   - Record dependency size, install path, startup time, model size, and packaging concerns.
   - Treat as a key validation item, not a Phase 1/2 blocker.

8. Produce the model capability matrix.
   - Fill `docs/phase0/model-matrix.md`.
   - Include license, output sample rate, peak memory, language support, cloning support, and recommended scenario.

## Outputs

- `docs/phase0/model-matrix.md`
- `docs/phase0/phase0-results.md`
- Minimal spike code or notes
- Decision: proceed to low-fidelity prototype, revise scope, or pause for missing dependencies
