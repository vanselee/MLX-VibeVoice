# Phase 0 Spike: mlx-audio-swift 集成计划

此文档记录 Phase 0 Spike 的目标、要求和验证标准。

---

## 🎯 Phase 0 目标

1. ✅ 验证 mlx-audio-swift 是否可以集成到我们的项目中
2. ✅ 验证可以在 Apple Silicon 上本地生成单句 TTS
3. ⏳ 验证性能（速度、内存）
4. ✅ 验证输出音频质量（bf16 通过）

---

## 📋 验收标准

### 必须通过 (Passing)

- [x] 可以编译项目，无链接错误
- [x] 可以导入并调用 mlx-audio-swift
- [x] 可以加载模型（使用 Qwen3-TTS-12Hz-0.6B-Base-bf16）
- [x] 可以将文本转为音频，保存 WAV 文件
- [x] 音频可以播放，声音清晰可懂
- [ ] 单次生成（~20 字）速度 < 5 秒
- [ ] 内存占用合理（< 1GB）

### 可选通过 (Nice to Have)

- [ ] 支持不同声音
- [ ] 支持调整语速
- [ ] 支持调整音量
- [ ] 支持参考音频克隆

---

## 🔧 集成步骤

### 步骤 1：添加 SPM 依赖 ✅

通过 Xcode GUI 手动添加 `mlx-audio-swift` SPM 依赖。

### 步骤 2：更新代码 ✅

更新 `MLXAudioService.swift`，适配真实 API：
- 添加 `import MLXLMCommon`
- 更新 `generate` 调用为完整参数版本
- 自定义 `createWAVData` 函数转换 [Float] 为 WAV

### 步骤 3：模型策略修正 ✅

移除 Soprano/Pocket/Kokoro/VyvoTTS 示例模型，只保留 Qwen3 系列：
- 默认模型：`mlx-community/Qwen3-TTS-12Hz-0.6B-Base-bf16`
- 添加本地缓存完整性检查，阻止自动下载

### 步骤 4：测试 ✅

使用 MLXTestView 进行真实测试。

**结论**:
- **8bit 模型**: ❌ 失败，输出为杂音
- **bf16 模型**: ✅ 通过，可生成可听语音
